## 3. АРХІТЕКТУРА РІШЕННЯ

### 3.1 Структура HTML (Семантична розмітка)

Акордеон побудований з використанням семантичних HTML елементів:

```html
<div class="accordion-container">
  <h3>
    <button id="acc-btn-1" aria-expanded="false" aria-controls="acc-panel-1">
      Секція 1: ...
    </button>
  </h3>
  <div id="acc-panel-1" role="region" aria-labelledby="acc-btn-1" hidden>
    <p>Вміст панелі...</p>
  </div>
</div>
```

**Ключові елементи:**

- **`<h3>`** — семантичний заголовок, вказує на ієрархію сторінки
- **`<button>`** — нативний інтерактивний елемент (підтримує Tab, Enter, Space за замовчуванням)
- **`role="region"`** — позначає панель як регіон (landmark) для навігації екранного диктора
- **`aria-expanded`** — вказує стан розгортання (true/false)
- **`aria-controls`** — зв'язує кнопку з панеллю, якою вона керує
- **`aria-labelledby`** — зв'язує панель з її заголовком
- **`hidden`** — приховує панель від HTML та дерева доступності

### 3.2 CSS Селектори на основі ARIA Атрибутів

```css
/* Показати панель, коли aria-expanded="true" */
[aria-expanded="true"] + [role="region"] {
  display: block;
  animation: slideDown 0.3s ease-out;
}

/* Приховати панель, коли aria-expanded="false" */
[aria-expanded="false"] + [role="region"] {
  display: none;
}

/* Фокус за WCAG 2.2 стандартами */
button[aria-expanded]:focus-visible {
  outline: 3px solid #ff6b6b;
  outline-offset: 2px;
}
```

**Переваги цього підходу:**

- CSS керує видимістю на основі ARIA стану
- Не потрібні додаткові CSS класи
- Семантично коректно

### 3.3 JavaScript Логіка

Реалізовано два основні механізми:

#### 3.3.1 Клік для перемикання панелі

```javascript
button.addEventListener("click", () => {
  const isExpanded = button.getAttribute("aria-expanded") === "true";
  const panelId = button.getAttribute("aria-controls");
  const panel = document.getElementById(panelId);

  // Зміна aria-expanded
  button.setAttribute("aria-expanded", !isExpanded);

  // Керування hidden атрибутом
  if (isExpanded) {
    panel.setAttribute("hidden", "");
  } else {
    panel.removeAttribute("hidden");
  }
});
```

#### 3.3.2 Клавіатурна навігація

```javascript
button.addEventListener("keydown", (event) => {
  const index = Array.from(buttons).indexOf(button);
  let newIndex;

  switch (event.key) {
    case "ArrowDown":
      event.preventDefault();
      newIndex = (index + 1) % buttons.length; // Циклічна навігація
      buttons[newIndex].focus();
      break;
    case "ArrowUp":
      event.preventDefault();
      newIndex = (index - 1 + buttons.length) % buttons.length;
      buttons[newIndex].focus();
      break;
    case "Home":
      event.preventDefault();
      buttons[0].focus();
      break;
    case "End":
      event.preventDefault();
      buttons[buttons.length - 1].focus();
      break;
  }
});
```

```html

  <body>
    <main>
      <h1>Доступний акордеон з ARIA та клавіатурною навігацією</h1>

        Вмикайте/вимикайте секції за допомогою кнопок. Навігація між
        заголовками: <kbd>Arrow Up</kbd> / <kbd>Arrow Down</kbd>, перехід до

      </p>

      <div class="accordion-container">
        <!-- 4 секції з ARIA атрибутами -->
        <h3>
          <button
            id="acc-btn-1"
            aria-expanded="false"
            aria-controls="acc-panel-1"
          >
            Секція 1: Загальна інформація про доступність
          </button>
        </h3>
        <div id="acc-panel-1" role="region" aria-labelledby="acc-btn-1" hidden>
          <p>
            Доступність в веб-розробці — це практика проєктування вебсайтів...
          </p>
        </div>

        <!-- Решта секцій... -->
      </div>
    </main>
    <script src="script.js"></script>
  </body>
</html>
```

### 4.2 styles.css

**Основні селектори:**

- `[aria-expanded="true"] + [role="region"]` — показати панель
- `[aria-expanded="false"] + [role="region"]` — приховати панель
- `button[aria-expanded]:focus-visible` — 3px blue outline (WCAG 2.2)
- Адаптивний дизайн для мобільних пристроїв
- Підтримка режиму з зниженою анімацією (`prefers-reduced-motion`)
- Підтримка високого контрасту (`prefers-contrast: more`)

### 4.3 script.js

**Функціонал:**

- Ініціалізація всіх кнопок при завантаженні сторінки
- Обробка подій `click` для перемикання `aria-expanded`
- Обробка подій `keydown` для навігації стрілками
- Циклічна навігація (на останній кнопці → перемикання на першу)
- Логування в консоль для дебагу

## 9. ВИСНОВКИ

### 9.1 Навчені навички

1. **ARIA атрибути**: `aria-expanded`, `aria-controls`, `aria-labelledby` мають вирішальне значення для передачі семантичної інформації допоміжним технологіям.

2. **Семантична верстка**: Використання нативних HTML елементів (`<button>`, `<h3>`) забезпечує вбудовану доступність та клавіатурну підтримку.

3. **Клавіатурна навігація**: Обробка подій `keydown` для навігації стрілками значно покращує UX користувачів, які не використовують мишу.

4. **CSS селектори ARIA**: Керування видимістю на основі ARIA атрибутів (замість класів) є більш семантично коректним та легше для підтримки.

5. **DevTools для тестування**: Chrome DevTools Accessibility Tree є інструментом, без якого неможливо розробляти доступні компоненти.

### 9.2 Практичне значення

Запропонована реалізація акордеона може бути використана як шаблон для:

- Розроблення інших складних інтерактивних компонентів (модальні вікна, меню, таби)
- Тренування команди розробників в сфері веб-доступності
- Аудиту існуючих проектів на відповідність WCAG стандартам

### 9.3 Відповідність стандартам

✓ **WCAG 2.2 Level AA** — Повна відповідність  
✓ **WAI-ARIA Best Practices** — Дотримано рекомендацій  
✓ **Section 508** (США) — Сумісна  
✓ **EN 301 549** (Європа) — Сумісна

### 9.4 Рекомендації для розширення

1. **Мультиплі відкриття**: Додати можливість відкривання кількох панелей одночасно (зняти обмеження "одна за раз")
2. **Анімації**: Розширити CSS анімації для плавнішого відкривання панелей
3. **Доступність у мобільній версії**: Тестувати з мобільними екранними дикторами
4. **Локалізація**: Підтримати кілька мов (сповіщення про стан може бути озвучено)
5. **API**: Додати JavaScript API для програматичного керування акордеоном

### 9.5 Висновок

Реалізація доступного акордеона демонструє, що при дотриманні практик WCAG та WAI-ARIA можна створити компоненти, які функціонують бездоганно для всіх користувачів — незалежно від того, використовують вони мишу, клавіатуру чи екранний диктор. Знання, набуті під час виконання цієї роботи, є основою для розроблення професійних, інклюзивних веб-інтерфейсів.
